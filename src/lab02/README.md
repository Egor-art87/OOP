# lab02

## Описание проекта

### Проект представляет собой реализацию класса-контейнера StudentCollection на языке Python, который предназначен для хранения и управления группой объектов Student (разработанного в лабораторной работе №1). Класс коллекции инкапсулирует внутренний список студентов и предоставляет методы для добавления, удаления, поиска, сортировки, фильтрации и итерации. В файле demo.py приведены примеры использования коллекции, демонстрирующие все её возможности.

## Мыслительный процесс создания класса коллекции

```txt 
При разработке класса StudentCollection я руководствовался следующими соображениями:

    Разделение ответственности

    Класс Student уже отвечает за одного студента. Для работы с группой студентов нужен отдельный класс-контейнер. Это соответствует принципу единственной ответственности: один класс – одна сущность (группа).

    Инкапсуляция внутреннего хранилища

    Внутренний список студентов (_items) объявлен защищённым (с префиксом _). Доступ к нему извне запрещён, все операции над списком выполняются через методы класса. Метод get_all() возвращает копию списка, чтобы внешний код не мог изменить коллекцию в обход правил.

    Контроль типа и уникальности

    Метод add() проверяет, что добавляемый объект является экземпляром Student (через isinstance). Также используется оператор in, который опирается на метод __eq__ класса Student, сравнивающий студентов по student_id. Это гарантирует отсутствие дубликатов (двух студентов с одинаковым идентификатором).

    Удобный интерфейс через магические методы

    Реализованы __len__ (позволяет писать len(collection)), __iter__ (позволяет использовать цикл for student in collection) и __getitem__ (поддерживает индексацию collection[0]). Благодаря этому коллекция ведёт себя как встроенный контейнер Python.

    Сортировка

    Метод sort(key, reverse) делегирует сортировку встроенному методу list.sort(), но с возможностью задать ключ (например, lambda s: s.name для сортировки по имени). Это даёт гибкость и не привязывает к конкретному полю.

    Фильтрация как логические операции

    Методы get_active() и get_by_course(course) возвращают новые экземпляры StudentCollection, содержащие только отфильтрованных студентов. Исходная коллекция при этом не изменяется. Это позволяет строить цепочки операций (например, group.get_active().sort(key=...)).

    Обработка ошибок

    Все публичные методы генерируют понятные исключения (TypeError, ValueError, IndexError) с сообщениями на русском языке. Это упрощает отладку и делает код более надёжным.

    Использование аннотаций типов

    Для самодокументирования кода и помощи IDE указаны типы параметров и возвращаемых значений (например, def add(self, student: Student) -> None).
```

### Создаётся коллекция StudentCollection. В неё добавляются пять объектов Student с уникальными семизначными ID. После каждого добавления выводится текущее количество студентов. Сценарий демонстрирует, что дубликаты не допускаются.

## Индексация и удаление по индексу
### Из коллекции удаляется студент, находящийся на позиции 1 (второй элемент), с помощью метода remove_at(1). После удаления количество студентов уменьшается, а порядок оставшихся сдвигается. Сценарий демонстрирует возможность удаления элемента без ссылки на сам объект.

## Сортировка коллекции
### Перед сортировкой в коллекцию возвращается ранее удалённый студент (чтобы показать сортировку на четырёх элементах). Затем применяются три вида сортировки: по имени (от А до Я), по возрасту (от младшего к старшему) и по курсу (по убыванию). Каждый раз выводится обновлённый порядок студентов. Сценарий показывает гибкость метода sort(key, reverse).

![sort_collection](/imeges/lab02/Сортировка.png)

## Фильтрация – активные студенты и студенты 2-го курса
### Сначала один из студентов (Иван Петров) переводится в неактивное состояние методом expel(). Затем вызывается get_active(), который возвращает новую коллекцию, содержащую только активных студентов. Выводится количество активных студентов и их имена. Аналогично get_by_course(2) возвращает коллекцию студентов, обучающихся на 2-м курсе. Исходная коллекция при этом не изменяется. Сценарий демонстрирует, что фильтрующие методы создают независимые копии.

## Заключение

### Данный проект демонстрирует основные принципы построения классов-контейнеров в Python: инкапсуляцию внутреннего хранилища, контроль типов и уникальности, реализацию магических методов для удобства использования, гибкую сортировку, фильтрацию с возвратом новых коллекций и грамотную обработку ошибок. Класс StudentCollection полностью готов к использованию в более крупных проектах, требующих управления группой студентов.
