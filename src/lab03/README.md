# lab03

## Описание проекта 

### Проект расширяет функциональность классов из предыдущих лабораторных работ. На основе базового класса `Student` (ЛР-1) построена иерархия из двух дочерних классов: `BakalavrStudent` (бакалавр) и `MagaStudent` (магистр). Класс-контейнер `StudentCollection` (ЛР-2) доработан для хранения объектов разных типов, наследующих от `Student`, и снабжён методами фильтрации по конкретным дочерним классам и по значению GPA.  В демонстрационном файле `demo.py` показано полиморфное поведение, единый интерфейс взаимодействия, фильтрация коллекции и работа бизнес-методов.

## При разработке я руководсвовался следующими принципами 

```txt
1. Переиспользование кода через наследование
Базовый класс Student из ЛР-1 содержит все общие атрибуты и методы (имя, возраст, GPA, курс, активность, перевод на следующий курс и т.д.). Дочерние классы BakalavrStudent и MagaStudent вызывают конструктор родителя через super().__init__(...), не дублируя уже существующую логику.

2. Расширение специфичными атрибутами и методами
- Бакалавр получает поля specialization (направление подготовки) и has_practice (флаг прохождения практики), а также методы complete_practice() и can_apply_master().
- Магистр получает поля supervisor (научный руководитель) и thesis_topic (тема диссертации), а также метод defend_thesis().

3. Переопределение методов для учёта особенностей уровня обучения
- Метод promote() ограничен 4‑мя курсами для бакалавра и 2‑мя для магистра.
- Метод __str__() дополнен выводом новых полей.
- Полиморфный метод calculate_scholarship() по‑разному рассчитывает стипендию: бакалавр получает надбавку за пройденную практику, магистр — за выбранную тему диссертации.
- Метод display_info() выводит полную информацию в едином стиле с указанием типа студента.

4. Полиморфизм без явной проверки типов
В демонстрационных сценариях и внутри коллекции не используются конструкции вида if type(obj) == ... для выбора поведения. Вместо этого у всех наследников есть единый интерфейс (display_info(), calculate_scholarship()), который вызывается у каждого объекта, а конкретная реализация определяется динамически.

5. Расширение коллекции методами фильтрации по типу
В класс StudentCollection добавлены:
- get_bakalavr_students() — возвращает новую коллекцию только с бакалаврами.
- get_maga_students() — возвращает новую коллекцию только с магистрами.
- get_by_gpa_threshold(min_gpa) — возвращает студентов с GPA не ниже заданного порога.
Внутри этих методов используется isinstance() — это допустимо в утилитарных методах контейнера и не нарушает полиморфизм бизнес-логики.

6. Обработка циклических импортов
Модули организованы так, что models.py импортирует base.py, а collection.py использует локальные импорты внутри методов, чтобы избежать циклов при подключении дочерних классов. Добавлен from __future__ import annotations для корректной работы аннотаций типов без необходимости глобального импорта.
```
